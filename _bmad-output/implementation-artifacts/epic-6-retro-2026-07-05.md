## What went well

1. **Pre-epic email provider spike (Kyle + team).** `pre-epic-6-email-provider-spike` completed before Story 6.1. Decision doc captured SDK patterns, idempotency conventions, 429 handling, and Vercel Hobby cron constraints. Story dev notes read like checklists because research was done upfront.

2. **Incremental email architecture.** 6.1 laid `sendWithRetry`, Resend singleton, and React Email templates; 6.2–6.5 reused patterns without reinventing. Idempotency keys (`invitation:`, `tuesday-digest:`, `reminder:`) consistent from day one.

3. **Review learnings propagated forward.** Story 6.2 CSRF missing → built into 6.3 spec. `sentAt > 0` guard and `preloadedData` pattern flowed 6.2 → 6.3 → 6.5 cron idempotency checks.

4. **Surgical deep links (6.4).** One login page server redirect + unit test; leveraged existing `(app)/layout.tsx` callbackUrl mechanism. Minimal scope, maximum impact.

5. **UX spec comparison finally closed (6.6).** Carried Epic 3 → 4 → 5; gap matrix recorded; league nav shell (DesktopAppBar + BottomNavigation), admin 2-column layout, responsive picks grid delivered.

6. **Epic 5 retro follow-through.** Provider spike, AdminPickOverrideDialog lint (fixed in 6.2), and 6.6 all landed. Sprint-status entries kept commitments alive.

7. **Strong test growth.** +27 tests; `league-nav-tabs` helper well-covered (14 tests).

---

## Challenges and friction

1. **Recurring deferred items across all stories.** TOCTOU race on concurrent sends, sequential per-member send timeouts, placeholder `from` domain appeared in every story's deferred section. Resend idempotency keys accepted as backstop; structural fixes deferred.

2. **Steady review patch rate (~4–6 per story).** Same as Epic 5. CSRF on mutating routes (6.2), idempotency edge cases, admin UI state bugs (6.6 email controls hidden during pre-season — caught in review).

3. **NFR32 Resend webhooks unassigned.** Delivery confirmation tracking still missing. Send logging exists; no webhook route.

4. **Hobby cron ±1h drift.** Eastern window guard prevents double-sends; negative drift can silently skip a week's send with no retry.

5. **Production go-live gap.** `noreply@yourdomain.com` placeholder remains in all senders. Domain verification and Vercel cron deployment deferred per project plan (production deploy after all epics).

6. **Manual testing gap.** Tuesday digest and reminder flows not yet manually verified (Kyle, retro close).
